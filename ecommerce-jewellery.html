Project Configuration Files

pom.xml

Maven Project Object Model file that defines the project structure, dependencies, and build configuration for the Spring Boot backend. This file specifies all the required libraries, plugins, and build settings needed to compile and package the Java application.

.mvn/wrapper/maven-wrapper.properties

Configuration file for the Maven Wrapper, which ensures all developers use the same Maven version for consistent builds. This helps avoid version conflicts between different development environments.

mvnw, mvnw.cmd

Maven wrapper scripts that allow you to run Maven commands without having Maven installed globally on your system. These scripts automatically download the correct Maven version specified in the wrapper properties file.

README.md

Project documentation file containing essential information about the project, including setup instructions, usage guidelines, and other important notes for developers and users.

Frontend Configuration Files

client/package.json

Node.js package configuration file that lists all frontend dependencies and scripts for the React application. This file defines the project name, version, and all required npm packages including React, Vite, Tailwind CSS, and other utilities.

client/package-lock.json

Automatically generated file that locks the versions of all dependencies to ensure consistent installations across different environments. This file should not be manually edited as it's maintained by npm.

client/index.html

The main HTML file that serves as the entry point for the React application. This file contains the root DOM element where the React application is mounted and basic HTML structure for the single-page application.

client/vite.config.js

Configuration file for Vite, the build tool used for the React frontend. This file specifies build options, development server settings, and plugin configurations that optimize the application for development and production.

client/tailwind.config.js

Configuration file for Tailwind CSS, defining custom styles, themes, and plugins used throughout the application. This file allows for customization of the utility-first CSS framework to match the project's design requirements.

client/eslint.config.js

Configuration file for ESLint, the JavaScript linter. This file defines coding standards, rules, and plugins that enforce consistent code style and help catch potential errors in the frontend codebase.

Global CSS stylesheet containing styles that apply to the entire application. This file typically includes base styles, global utilities, and any CSS resets or normalizations needed for consistent rendering across browsers.

Frontend Components and Pages

client/src/components/AdminLayout.jsx

Layout component specifically designed for admin pages. This component provides a consistent navigation structure, header, and footer for all administrative interfaces, ensuring a uniform experience across different admin sections.

client/src/hooks/useCheckoutGuard.js

Custom React hook that implements protection for checkout routes. This hook ensures that users cannot access checkout pages without proper authorization or without items in their cart, improving the security and user experience of the checkout process.

client/src/utils/storage.js

Utility functions for handling local storage operations. This file contains helper functions for saving and retrieving data from the browser's local storage, such as user preferences, authentication tokens, or temporary cart data.

client/src/pages/Home.jsx

The landing page component that serves as the main entry point for the e-commerce site. This page typically displays featured products, promotions, navigation options, and other key elements to engage visitors and guide them through the shopping experience.

client/src/pages/Product.jsx

Component that displays detailed information about a single product. This page includes product images, description, specifications, pricing, availability, and add-to-cart functionality, providing customers with all the information they need to make a purchase decision.

client/src/pages/Cart.jsx

Shopping cart page that shows all items a user has added to their cart. This page allows customers to review their selected products, modify quantities, remove items, and proceed to checkout, with clear displays of prices, totals, and any applicable discounts.

client/src/pages/Checkout.jsx

Checkout page where users review their order, enter shipping information, and select payment methods. This multi-step process typically includes order summary, shipping address form, payment details, and final confirmation before completing the purchase.

client/src/pages/Account.jsx

User account management page where customers can view and update their personal information. This page typically includes order history, profile settings, address book, payment methods, and other account-related features to help users manage their relationship with the e-commerce platform.

client/src/pages/Login.jsx

User authentication page where customers can log in to their existing accounts. This page includes a login form with email/username and password fields, as well as options for password recovery and account registration for new users.

client/src/pages/Signup.jsx

User registration page where new customers can create an account. This page collects necessary information such as name, email, password, and other relevant details to set up a new user profile in the system.

client/src/pages/ForgetPassword.jsx

Password recovery page where users can request a password reset if they've forgotten their credentials. This page typically includes a form to enter the registered email address and initiates the password reset process by sending a verification link.

client/src/pages/Payment.jsx

Payment processing page where users enter their payment information to complete a purchase. This secure page handles sensitive financial data and integrates with payment gateways to process transactions while maintaining compliance with security standards.

client/src/pages/Success.jsx

Order confirmation page displayed after a successful purchase. This page shows order details, confirmation state and final bill with a confetti!

client/src/pages/Category.jsx

Page that displays products filtered by a specific category. This page helps users browse products organized by type, style, or other classification, with options to further refine the selection using filters and sorting options.

client/src/pages/Collections.jsx

Page showcasing curated collections of products, often grouped by theme, season, or special promotions. This page highlights specially selected product groupings to inspire customers and encourage exploration of related items.

client/src/pages/Contact.jsx

Contact page with a form for customers to send messages to customer support or request information. This page provides multiple ways for users to get in touch with the business, including contact forms, email addresses, phone numbers, and possibly live chat options.

client/src/pages/Delivery.jsx

Information page detailing shipping options, delivery times, and related policies. This page helps customers understand their delivery choices, estimated arrival times, shipping costs, and any special handling instructions for their orders.

Admin Pages

client/src/pages/admin/AdminDashboard.jsx

Main dashboard for administrators, providing an overview of key metrics and quick access to management tools. This page typically shows summary statistics, recent activity, and shortcuts to various administrative functions.

client/src/pages/admin/Dashboard.jsx

Alternative or supplementary dashboard component for administrative functions. This might provide additional views or specialized information for different types of administrative users.

client/src/pages/admin/ProductsAdmin.jsx

Administrative interface for managing products in the catalog. This page allows administrators to add new products, edit existing ones, update inventory levels, set pricing, and manage product attributes and categories.

client/src/pages/admin/UsersAdmin.jsx

Administrative interface for managing user accounts. This page provides tools to view customer information, manage user permissions, handle account issues, and perform other user-related administrative tasks.

client/src/pages/admin/OrdersAdmin.jsx

Administrative interface for viewing and managing customer orders. This page allows staff to process orders, update order statuses, handle returns or exchanges, and manage other aspects of the order fulfillment process.

client/src/pages/admin/CartsAdmin.jsx

Administrative interface for viewing active shopping carts. This tool helps with abandoned cart recovery, customer support inquiries about cart contents, and monitoring shopping behavior patterns.

client/src/pages/admin/CouponsAdmin.jsx

Administrative interface for creating and managing discount coupons and promotional codes. This page allows marketing teams to create new promotions, set discount rules, track coupon usage, and analyze the effectiveness of different promotional campaigns.

Backend Source Files

src/main/java/com/ecommerce/jewelleryMart/JewelleryMartApplication.java

The main application class containing the main() method that boots up the Spring Boot application. This class initializes the Spring application context and serves as the entry point for the backend service.

src/main/java/com/ecommerce/jewelleryMart/config/SecurityConfig.java

Spring Security configuration class that defines authentication and authorization rules. This file configures security settings including user authentication, role-based access control, CSRF protection, CORS policies, and other security-related aspects of the application.

Backend Controllers

src/main/java/com/ecommerce/jewelleryMart/controller/AuthController.java

REST controller handling user authentication operations. This controller manages endpoints for user login, registration, logout, password recovery, and other authentication-related functions, typically working with JWT or session-based authentication mechanisms.

src/main/java/com/ecommerce/jewelleryMart/controller/UserController.java

REST controller managing user-related operations. This controller handles endpoints for user profile management, account updates, user information retrieval, and other user-specific functions that don't relate directly to authentication.

src/main/java/com/ecommerce/jewelleryMart/controller/ProductController.java

REST controller handling product-related operations. This controller provides endpoints for CRUD operations on products, including creating new products, updating existing ones, retrieving product information, searching and filtering products, and managing product inventory.

src/main/java/com/ecommerce/jewelleryMart/controller/CartController.java

REST controller managing shopping cart operations. This controller handles endpoints for adding items to carts, removing items, updating quantities, retrieving cart contents, and other cart-related functions that support the shopping experience.

src/main/java/com/ecommerce/jewelleryMart/controller/OrderController.java

REST controller handling order processing. This controller provides endpoints for creating new orders, retrieving order history, updating order statuses, processing returns or cancellations, and other order management functions.

src/main/java/com/ecommerce/jewelleryMart/controller/CheckoutController.java

REST controller managing the checkout process. This controller handles endpoints for order review, payment processing, order confirmation, and other steps in the checkout workflow that bridge the gap between shopping and order fulfillment.

src/main/java/com/ecommerce/jewelleryMart/controller/CouponController.java

REST controller handling coupon-related operations. This controller provides endpoints for validating coupons, applying discounts to orders, creating new promotional codes, and managing the lifecycle of discount offers within the system.

Backend Models

src/main/java/com/ecommerce/jewelleryMart/model/User.java

JPA entity class representing a user account. This class defines the structure of user data stored in the database, including fields for authentication credentials, personal information, roles, and other user attributes, along with relationships to other entities like orders and carts.

src/main/java/com/ecommerce/jewelleryMart/model/Product.java

JPA entity class representing a product in the catalog. This class defines the structure of product data including fields for product name, description, price, inventory levels, categories, attributes, images, and other product-related information needed for display and management in the e-commerce system.

src/main/java/com/ecommerce/jewelleryMart/model/Cart.java

JPA entity class representing a shopping cart. This class defines the structure of cart data including the items selected by a user, quantities, prices at time of addition, and other cart-specific information, typically with relationships to both user and product entities.

src/main/java/com/ecommerce/jewelleryMart/model/Order.java

JPA entity class representing a customer order. This class defines the structure of order data including order items, quantities, prices, shipping information, payment details, order status, and other information needed to process and fulfill customer purchases.

src/main/java/com/ecommerce/jewelleryMart/model/Coupon.java

JPA entity class representing a discount coupon. This class defines the structure of coupon data including the coupon code, discount value or percentage, validity period, usage restrictions, and other attributes that determine how the coupon can be applied to orders.

Backend Repositories

src/main/java/com/ecommerce/jewelleryMart/repository/UserRepository.java

Spring Data JPA repository interface for User entities. This interface extends JpaRepository to provide CRUD operations and custom query methods for user data, enabling database access and manipulation of user records without requiring boilerplate implementation code.

src/main/java/com/ecommerce/jewelleryMart/repository/ProductRepository.java

Spring Data JPA repository interface for Product entities. This interface provides database access and query methods specific to product data, supporting operations like product searches, filtering, and inventory management through method naming conventions and custom queries.

src/main/java/com/ecommerce/jewelleryMart/repository/CartRepository.java

Spring Data JPA repository interface for Cart entities. This interface offers database operations tailored to shopping cart data, including methods to retrieve, update, and manage cart contents with relationships to both users and products.

src/main/java/com/ecommerce/jewelleryMart/repository/OrderRepository.java

Spring Data JPA repository interface for Order entities. This interface provides specialized database operations for order data, supporting complex queries related to order processing, fulfillment, and historical analysis of customer purchases.

src/main/java/com/ecommerce/jewelleryMart/repository/CouponRepository.java

Spring Data JPA repository interface for Coupon entities. This interface enables database operations specific to coupon management, including validation, retrieval, and tracking of promotional codes and their usage within the system.

Backend Configuration

src/main/resources/application.properties

Spring Boot configuration file containing application properties. This file defines various settings for the application including database connection details, server configuration, security settings, logging configuration, and other environment-specific parameters that control the behavior of the Spring Boot application.

Project Architecture Note: This application follows a modern full-stack architecture with clear separation between frontend and backend concerns. The React frontend communicates with the Spring Boot backend through RESTful API endpoints. The backend implements a layered architecture with controllers handling HTTP requests, services containing business logic, repositories managing data access, and models representing the domain entities. This structure promotes maintainability, scalability, and testability of the application.
